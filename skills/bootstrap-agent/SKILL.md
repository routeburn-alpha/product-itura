# Bootstrap Agent

Fix or refresh the current agent workspace. Idempotent — safe to run multiple times.

## Step 1: Check settings

Read `.claude/settings.local.json`, `.codex/config.toml`, and `.codex/environments/environment.toml` if present. If `AGENT_NAME`, `X-Agent-Name`, or `AGENT_PORT` are missing, ask the user what agent name and port to use.

## Step 2: Ensure Claude Code settings exist

If missing, write `.claude/settings.local.json`:

```json
{
  "env": {
    "AGENT_NAME": "<name>",
    "AGENT_PORT": "<port>"
  }
}
```

Also write or refresh `CLAUDE.local.md`:

```markdown
# Agent Identity

- **Name**: <name>
- **Port**: <port> (Vite dev server)
```

## Step 3: Ensure Codex settings exist

If missing, write `.codex/config.toml`. Copy the Studio URL/header shape from `.mcp.json`, set `X-Agent-Name` to the agent name, and leave the bearer token placeholder for the user to fill in:

```toml
[mcp_servers.studio-ai]
url = "https://app.routeburn.org/api/mcp"

[mcp_servers.studio-ai.http_headers]
Authorization = "Bearer <studio-ai-token>"
X-Agent-Name = "<name>"
```

If `.codex/config.toml` already exists with a real token, preserve the `Authorization` value and only add or correct `X-Agent-Name`.

Also ensure `.codex/environments/environment.toml` exists for local dev-server convenience:

```toml
# THIS IS AUTOGENERATED. DO NOT EDIT MANUALLY
version = 1
name = "<worktree name>"

[setup]
script = '''
export AGENT_NAME=<name>
export AGENT_PORT=<port>
'''
```

## Step 4: Sync with main

Stash local changes if any, rebase onto origin/main, restore stash:

```bash
git stash  # only if dirty
git fetch origin main && git rebase origin/main
git stash pop  # only if stashed
```

If there are rebase conflicts, stop and ask the user.

## Step 5: Install dependencies

Dependencies live under `app/`:

```bash
cd app
rm -rf node_modules && npm install
```

## Step 6: Verify the app builds and tests pass

```bash
cd app
npm run check
npm test
```

Optionally start the dev server to confirm it boots (don't leave it running in the background):

```bash
cd app
PORT=$AGENT_PORT npm run dev
```

## Step 7: Instruct user to restart their client

After all steps complete, tell the user:

> Bootstrap complete! You need to **restart Claude Code or Codex** for the environment variables and MCP configuration to take effect. Exit this session and start a new one from this worktree.

This is required because both clients read environment variables and MCP server config at startup.

---

## Troubleshooting

- **Rebase conflicts**: ask the user
- **npm install failure**: `rm -rf app/node_modules app/package-lock.json` and retry (only nuke the lockfile as a last resort and call it out to the user)
- **`AGENT_NAME environment variable is required` from Studio MCP**: verify `.codex/config.toml` sends `X-Agent-Name` under `[mcp_servers.studio-ai.http_headers]`, then restart Codex
- **Port already in use**: another agent is probably running on the same port — bump `AGENT_PORT` in `.claude/settings.local.json` and `.codex/environments/environment.toml`
