# Create Agent

Create a new agent workspace using a git worktree. Does the bare minimum so `/bootstrap-agent` can run in the new directory. Seed both supported client paths:

- Claude Code: `.claude/settings.local.json`
- Codex: `.codex/config.toml` plus `.codex/environments/environment.toml`

## Agent naming

Ask the user what they want to name the agent identifier (e.g., "kevin", "felix", "alice"). The dev-server port is derived from the agent number: `port = 5173 + agent number` (5173 is the Vite default).

---

## Step 1: Find next available agent number

Derive the base directory and worktree prefix from the current repo, so new agents are siblings of this worktree and the prefix works in any fork of the template:

```bash
REPO_ROOT=$(git rev-parse --show-toplevel)
BASE_DIR=$(dirname "$REPO_ROOT")
REPO_NAME=$(basename "$REPO_ROOT")
ls -d "$BASE_DIR/$REPO_NAME-"* 2>/dev/null | sed "s|.*$REPO_NAME-||" | grep -oE '^[0-9]+' | sort -n | tail -1
```

Next agent number = highest existing number + 1. If none exist, start at 1.

## Step 2: Ask for agent name

If the user already provided a name, use it. Otherwise, ask: "What do you want to name this agent?"

The port is `5173 + NEXT_NUM`.

## Step 3: Create worktree

The directory name includes both the number and the identifier: `$REPO_NAME-$NEXT_NUM-$AGENT_NAME`

```bash
git worktree prune
git worktree add -b agent/$AGENT_NAME "$BASE_DIR/$REPO_NAME-$NEXT_NUM-$AGENT_NAME" main
```

## Step 4: Create Claude Code settings

```bash
mkdir -p "$BASE_DIR/$REPO_NAME-$NEXT_NUM-$AGENT_NAME/.claude"
```

Write `.claude/settings.local.json` (this is what Claude Code/bootstrap reads to know the agent's identity):

```json
{
  "env": {
    "AGENT_NAME": "<agent name>",
    "AGENT_PORT": "<port number>"
  }
}
```

## Step 5: Create Codex Studio MCP config

```bash
mkdir -p "$BASE_DIR/$REPO_NAME-$NEXT_NUM-$AGENT_NAME/.codex"
```

Write `.codex/config.toml`. Copy the Studio URL/header shape from `.mcp.json`, set `X-Agent-Name` to this agent, and leave the bearer token as a placeholder for the user to fill in:

```toml
[mcp_servers.studio-ai]
url = "https://app.routeburn.org/api/mcp"

[mcp_servers.studio-ai.http_headers]
Authorization = "Bearer <studio-ai-token>"
X-Agent-Name = "<agent name>"
```

Also write the local Codex environment file for dev-server convenience:

```bash
mkdir -p "$BASE_DIR/$REPO_NAME-$NEXT_NUM-$AGENT_NAME/.codex/environments"
cat > "$BASE_DIR/$REPO_NAME-$NEXT_NUM-$AGENT_NAME/.codex/environments/environment.toml" <<EOF
# THIS IS AUTOGENERATED. DO NOT EDIT MANUALLY
version = 1
name = "$REPO_NAME-$AGENT_NAME"

[setup]
script = '''
export AGENT_NAME=$AGENT_NAME
export AGENT_PORT=$AGENT_PORT
'''
EOF
```

## Step 6: Report and provide next steps

Tell the user:
```
New agent created at $BASE_DIR/$REPO_NAME-N-NAME

Claude Code path:
1. Open a new terminal
2. cd $BASE_DIR/$REPO_NAME-N-NAME
3. claude
4. Run /bootstrap-agent to install dependencies and verify tests

Codex path:
1. Open .codex/config.toml in the new worktree
2. Replace <studio-ai-token> with your Studio bearer token
3. Confirm X-Agent-Name is set to NAME
4. Open a new terminal
5. cd $BASE_DIR/$REPO_NAME-N-NAME
6. codex
7. Run /bootstrap-agent to install dependencies and verify tests
```
